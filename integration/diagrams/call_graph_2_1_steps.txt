- Draw a hand-made function call graph showing calls within and across modules.
- One node = one function.
- Arrow direction = caller -> callee.

Step 5: Make cross-module calls visually clear
- Draw each module in a separate rectangle.
- Use black arrows for same-module calls.
- Use blue arrows for cross-module calls.
- Put a small legend at top-right:
  - black = intra-module
  - blue = inter-module

Step 6: Keep the graph readable
- Place modules roughly in layers:
  - Top: registration, crew_management, inventory
  - Middle: race_management, mission_planning, maintenance, results
  - Bottom: reporting
- Keep arrows mostly left-to-right or top-to-bottom.
- Avoid arrow overlap as much as possible.

Step 7: Add title and caption
Title:

Caption line under figure:
- Each node is a single function. Arrows indicate caller to callee. Blue arrows denote cross-module function calls.

Step 8: Verify before submission
Checklist:
- Every function in module list is shown as a node.
- Every key call edge listed above is drawn.
- Cross-module calls are clearly distinguishable.
- Image is clear and readable in report.
